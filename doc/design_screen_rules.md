# 画面設計書記述ルール

## タイトル

* 形式が`\[画面ID] 画面名`になっていること

## 変更履歴セクション

* テーブルは`<div class="log">`の要素に囲まれていること

### 項目「No」

* 空欄ではないこと

### 項目「変更日」

* 空欄ではないこと

### 項目「変更セクション」

* 空欄ではないこと

### 項目「変更項目」

* 空欄ではないこと
* （ーは許容）

### 項目「変更者」

* 空欄ではないこと

## 画面入出力項目セクション

* テーブルは`<div class="ui-element">`の要素に囲まれていること

### 項目「No」

* 空欄ではないこと

### 項目「項目名」

* 空欄ではないこと

### 項目「イベント」

* 空欄ではないこと
* イベントがない場合は`-`と記述すること

### 項目「フォームの種類」

* 以下の値以外が入っていないこと

  * データ表示
  * リンク表示
  * 画像表示
  * テキスト
  * メールアドレス
  * テキストエリア
  * パスワード
  * ラジオボタン
  * チェックボックス
  * セレクトボックス
  * マルチプルセレクト
  * ファイル選択（画像含む）
  * 日時（カレンダー選択式）
  * 日付（カレンダー選択式）
  * 時刻（カレンダー選択式）
  * ボタン（画像ボタン含む）

### 項目「必須」

* 空欄ではないこと
* 必須ではない場合は`-`と記述すること

### 項目「桁数」

* 制限がない場合は`-`と記述すること
* `最小`と`最大`についての情報を記述すること
* 以下の形式であること

  * 最大：{数字}
  * 最小：{数字}

### 項目「制約」

* 制限がない場合は`-`と記述すること
* 以下の形式であること

  * {制約}：{内容}
  * 例 `形式：メールアドレス`

### 項目「備考」

* 空欄ではないこと
* 備考がない場合は`\&nbsp;`と記述すること

## イベント処理概要セクション

### イベントタイトル

* イベントNoの値は、画面入出力項目の`イベント`に`○`がついている項目のNoと一致すること（初期表示は`No.0`であること）
* 画面入出力項目のNo3に`○`がついている場合は、`No.3 イベントタイトル`

### INPUT

* テーブルは`<div class="io-item">`の要素に囲まれていること
* 処理に影響を及ぼす項目を記述すること
* 影響がない場合は記述不要

### OUTPUT

* テーブルは`<div class="io-item">`の要素に囲まれていること
* レスポンス（画面表示やjsonレスポンス等）を記述すること
* また、`クッキー`や`セッション`に項目を新規設定した場合も記述すること

### 処理

#### 処理内容の記述形式

* 処理番号を使って記述すること
* 1階層目の処理では、処理の目的を記述すること
* 例 `1.商品情報を取得する`、`2.入力フォームの値をチェックする`、`フォーム項目の制御`

#### OUTPUTの記述形式

* `→`の後には、OUTPUTの内容を記述すること
* 形式は`OUTPUT先：OUTPUT内容`
* 例 `→画面表示：バリデーションエラーメッセージ表示` ※OUTPUTにバリデーションエラーメッセージ表示と記述されている場合
* セッションなどで複数OUTPUT内容がある場合は`OUTPUT先：OUTPUT内容1,OUTPUT内容2`という記述をする
* ※ 後続の処理がない場合は`（処理終了）`と追加で記述すること

#### APIコールの記述形式

* APIコールする処理は`システム名「API名」`という形式で表現すること
* 例 `ペッツ会員DB「会員仮登録API」をコールする`
* APIコールする処理には、設定するリクエストパラメータについても記述すること
例

```
  リクエストパラメータ
  　キー名 = 設定する値
  　email = 入力フォームのメールアドレス
  ```

#### APIレスポンスの記述形式

* APIレスポンスは`レスポンスの項目名`という形式で表現する
* （`2-1のレスポンスの項目名`など処理番号を追加することも可能）
* 例 レスポンスの`セッションID`

#### DBデータ取得の記述形式

* DBから値を取得する場合は、以下のように記述する（取得条件がない場合は省略可能）
例

```
  対象テーブル
  　商品マスター
  取得条件
  　商品マスター.区分=GETパラメーターの本誌タイプ
  　商品マスター.適用開始期間 >= 現在日時
  　現在日時 < 商品マスター.適用終了期間
  ```

#### DBデータの記述形式

* DB項目は`テーブル名.カラム名`という形式で表現する
* （`2-1のテーブル名.カラム名`など処理番号を追加することも可能）
* 例 `キャンペーンマスター.キャンペーンコード`

## エラー表示ルール

### 形式

* 原則として、ブラウザ標準の `alert()` は使用禁止とする。
* エラーメッセージは、画面内の適切な位置（主にコンテンツの上部）にインラインバナー形式で表示すること。
* バナーにはエラー内容を明記し、ユーザーが内容を確認した後に手動で閉じられる「×」ボタンを設置すること。

### 持続性

* 非同期処理の完了（Loading状態の終了）に伴ってエラーメッセージが自動的に消えないように制御すること。
* ユーザーが明示的に閉じるか、再度処理を正常に実行するまで表示を維持すること。
