# エンジニアリング動作確認規定

## 目的
頻繁に発生するインポートエラー、サーバー起動失敗、およびUIのデグレを未然に防ぎ、常に高品質で「動く状態」のコードを提供することを目的とする。

## 1. サーバー起動および環境の初期確認
コード変更後、またはタスク完了前に、必ず以下の確認を行うこと。

*   **バックエンドサーバーの起動確認**:
    - `python -m uvicorn backend.main:app` 等のコマンドでサーバーが正常に起動すること。
    - 起動ログにおいて `ImportError`, `ModuleNotFoundError` が発生していないことを目視で確認すること。
*   **フロントエンドサーバーの起動確認**:
    - `npm run dev` でエラーなく起動し、ブラウザからアクセス可能であることを確認すること。

## 2. 機能検証（スモークテスト）
変更箇所に関連する主要なユースケースについて、以下の検証を実施すること。

*   **APIエンドポイントの検証**:
    - 変更したAPIが意図したJSONレスポンスを返すこと。
*   **画面遷移と表示の検証**:
    - 変更した画面が表示崩れなくロードされること。
    - 特に動的な要素（グラフ、インラインバナー等）が正しく描画されること。
*   **異常系の検証**:
    - バリデーションエラーやAPIエラー時に、適切なエラーメッセージが表示されること。
    - **ルール準拠確認**: エラー表示は「永続的」であり、手動で閉じられる形式であること。

## 3. 静的解析と品質チェック
*   **型チェック・構文チェック**:
    - 変更したファイルに対して静的解析ツール（Pyre, TypeScriptコンパイラ等）を実行し、エラーがないことを確認すること。
*   **未使用インポートの削除**:
    - コードの可読性と保守性を維持するため、未使用のインポートや変数は直ちに削除すること。

## 4. 証跡の記録
*   UIの変更や重要な機能更新の際は、ブラウザサブエージェント等を用いてスクリーンショットまたは動画を記録し、プルリクエストやウォークスルーに添付すること。
