# 画面設計書 (Screen Specifications)

本ドキュメントは、EDINETデータ分析アプリケーションのフロントエンド画面における設計・仕様をまとめたものです。

## 概要

本アプリケーションは、React と React Router を用いて構築されたSPA (Single Page Application) です。ナビゲーションバーを共通コンポーネントとして持ち、各機能はページコンポーネントとして分割されています。

---

## 1. 共通レイアウト

### 1.1 ナビゲーションバー (Navbar)
*   **役割**: アプリケーション全体のグローバルナビゲーション。
*   **配置**: 画面上部 (ヘッダー部分)
*   **要素**:
    *   ロゴ / アプリケーション名 (ダッシュボードへのリンク)
    *   リンクメニュー: ダッシュボード、分析手法、ポートフォリオ
    *   ユーザー情報: ログインユーザー名、ログアウトボタン

---

## 2. 画面詳細

### [SCR001] [ダッシュボード画面](screens/dashboard.md)
*   **パス**: `/dashboard`
*   **役割**: ログイン直後に表示されるホーム画面。主要機能へのショートカットと、最近のデータ更新状況を提供する。

### [SCR002] [銘柄詳細画面](screens/stock_detail.md)
*   **パス**: `/stocks/:code`
*   **役割**: 特定の企業（銘柄）の詳細なハイライトと財務データを視覚的に表示する。

### [SCR003] [分析手法画面](screens/analysis_methods.md)
*   **パス**: `/analysis`
*   **役割**: ユーザー独自のスクリーニング条件 (分析手法) を作成・管理し、実行結果を確認する。

### [SCR004] [ポートフォリオ一覧画面](screens/portfolios.md)
*   **パス**: `/portfolios`
*   **役割**: ユーザーが作成したポートフォリオのグループを管理する。

### [SCR005] [ポートフォリオ詳細画面](screens/portfolio_detail.md)
*   **パス**: `/portfolios/:id`
*   **役割**: 選択したポートフォリオに含まれる具体的な銘柄(Portfolio Items)を一覧表示・管理する。

### [SCR006] [ログイン画面](screens/login.md)
*   **パス**: `/login`
*   **役割**: 既存ユーザーのログイン。

### [SCR007] [新規登録画面](screens/register.md)
*   **パス**: `/register`
*   **役割**: 新規ユーザーの登録。
