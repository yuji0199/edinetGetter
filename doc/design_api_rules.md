# API設計書記述ルール

## タイトル

* 形式が`\[API ID] 概要`になっていること
* 例：`[API001] ユーザーログイン`

## 基本情報

* 以下の項目を記述すること
  * **Method**: GET / POST / PUT / DELETE 等
  * **Path**: `/api/v1/auth/login` 等
  * **認証**: 必要 / 不要

## 変更履歴セクション

* テーブルは`<div class="log">`の要素に囲まれていること
* 項目「No」「変更日」「変更セクション」「変更項目」「変更者」を記述すること
* 空欄は作成しないこと

## リクエストパラメータセクション

* テーブルは`<div class="api-request">`の要素に囲まれていること
* 以下の項目を記述すること
  * **No**
  * **場所**: Header / Query / Path / Body
  * **項目名**
  * **型**: String / Number / Boolean / Object / Array
  * **必須**: ○ / -
  * **備考**: 制約（桁数、形式等）やデフォルト値を記述

## レスポンスセクション

* テーブルは`<div class="api-response">`の要素に囲まれていること
* 成功時（200系）と失敗時（400/500系）に分けて記述すること
* 以下の項目を記述すること
  * **No**
  * **項目名**
  * **型**
  * **備考**

## 処理詳細セクション

* テーブルは`<div class="api-logic">`の要素に囲まれていること
* 以下の形式で処理フローを記述すること
  1. 処理の目的・概要
  2. バリデーションチェック
  3. DB操作、外部APIコール等の詳細
* DB項目は`テーブル名.カラム名`の形式で記述すること
* OUTPUT（レスポンス）は `→項目名: 内容` の形式で記述すること
