## イベント処理概要

### No.0 初期表示

**INPUT**
<div class="io-item">

| 項目名 | 備考 |
| :--- | :--- |
| 認証トークン | クッキーより取得 |

</div>

**OUTPUT**
<div class="io-item">

| 項目名 | 備考 |
| :--- | :--- |
| 最近の同期書類リスト | 画面中央のテーブルに表示 |
| ユーザー情報 | ヘッダーに表示 |

</div>

**処理**
1. 最新の同期書類を取得する
   - バックエンド「最近の書類取得API」をコールする
   →画面表示：最近同期された書類一覧にデータを反映
2. ユーザー情報を取得する
   - バックエンド「現在のユーザー取得API」をコールする
   →画面表示：ユーザー名ラベルに反映（処理終了）

### No.6 ログアウトボタン押下

**処理**
1. ログアウト処理を実行する
   - 認証トークンを削除する
   →画面表示：ログイン画面へ遷移（処理終了）

### No.9 同期ジョブを開始ボタン押下

**INPUT**
<div class="io-item">

| 項目名 | 備考 |
| :--- | :--- |
| 同期対象日 | 項目No.8の値 |

</div>

**処理**
1. 同期ジョブのリクエストを送信する
   - バックエンド「EDINET同期API」をコールする
     リクエストパラメータ
     　date = 同期対象日の値
   →画面表示：処理メッセージ表示エリアに「同期処理を開始しました（バックグラウンドで実行中）」と表示（処理終了）
